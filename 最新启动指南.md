# IFM-Service 应用程序最新启动指南

## 一、检查MySQL服务状态

1. 按下 `Win + X` 组合键，选择 **Windows PowerShell（管理员）**
2. 执行以下命令检查MySQL服务状态：
   ```powershell
   sc query MySQL80
   ```
3. 如果显示服务状态为 `RUNNING`，跳过下一步
4. 如果服务未运行，执行以下命令启动服务：
   ```powershell
   net start MySQL80
   ```

## 二、创建数据库

1. 打开Windows PowerShell（管理员）
2. 执行以下命令连接到MySQL服务器：
   ```powershell
   "C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p
   ```
   *注意：如果MySQL安装路径不同，请调整为实际路径*
3. 系统会提示输入密码，输入 `@Zyain204918` 并回车
4. 成功连接后，执行以下命令创建数据库：
   ```sql
   CREATE DATABASE ifm_service DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
5. 执行以下命令确认数据库创建成功：
   ```sql
   SHOW DATABASES;
   ```
6. 看到 `ifm_service` 出现在列表中，表示创建成功
7. 输入 `exit` 退出MySQL命令行

## 三、构建项目

1. 打开Windows PowerShell
2. 导航到项目根目录：
   ```powershell
   cd d:\02_Academic\ifm-service
   ```
3. 执行Maven构建命令：
   ```powershell
   mvn clean package -DskipTests
   ```
4. 等待构建完成，看到 `BUILD SUCCESS` 表示构建成功

## 四、启动应用程序

1. 在项目根目录下执行以下命令：
   ```powershell
   java -jar target/ifm-service-0.0.1-SNAPSHOT.jar
   ```
2. 等待应用程序启动，观察控制台输出
3. 如果看到类似以下内容，表示应用程序启动成功：
   ```
   2025-xx-xx xx:xx:xx.xxx  INFO xxxx --- [main] o.s.b.w.embedded.tomcat.TomcatWebServer  : Tomcat started on port(s): 8080 (http) with context path ''
   2025-xx-xx xx:xx:xx.xxx  INFO xxxx --- [main] c.z.y.ifmservice.IfmServiceApplication   : Started IfmServiceApplication in x.xxx seconds (process running for x.xxx)
   ```

## 五、验证应用程序

1. 打开浏览器，访问以下地址：
   ```
   http://localhost:8080/swagger-ui.html
   ```
2. 如果能看到Swagger UI界面，表示应用程序已成功启动并运行

## 六、常见问题排查

### 1. MySQL服务无法启动
- 检查MySQL安装是否完整
- 查看MySQL错误日志（通常位于 `C:\ProgramData\MySQL\MySQL Server 8.0\Data\` 目录下）

### 2. 无法连接到MySQL服务器
- 确认MySQL服务正在运行
- 确认root密码输入正确
- 确认MySQL安装路径是否正确

### 3. 数据库创建失败
- 确认MySQL用户有创建数据库的权限
- 确认数据库名称拼写正确

### 4. 应用程序启动失败
- 检查控制台输出的错误信息
- 确认数据库连接配置正确
- 确认数据库已成功创建

## 七、关闭应用程序

1. 在启动应用程序的PowerShell窗口中，按下 `Ctrl + C` 组合键
2. 等待应用程序优雅关闭
